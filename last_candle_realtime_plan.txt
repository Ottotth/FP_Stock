FP_Stock - 最後一根 K 棒即時更新計劃（延後實作）

文件用途：
- 作為後續實作的規劃文件。
- 目前先不要實作。

============================================================
1) 目標
============================================================
建立「最後一根 K 棒即時變動」機制，讓蠟燭圖最後一根在市場開盤期間可持續更新（OHLC 變化），而不需要每幾秒就整包重抓歷史資料。

預期使用者體驗：
- 進入圖表頁：歷史資料只載入一次。
- 留在頁面時：只持續更新最後一根 K。
- 切換股票/週期：清理舊狀態後重新初始化。
- 重新整理頁面：可接受重新載入一次歷史資料。

============================================================
2) 資料策略（建議）
============================================================
採用兩條資料流：
A) 歷史 K 棒（穩定、可持久化）
B) 即時當前 K 棒（尚未收線的 bucket）

歷史資料維持既有 API。
另外新增專門給即時最後一根 K 的 API。

原因：
- 避免重複查詢高成本歷史資料。
- 降低圖表閃爍與傳輸量。
- 清楚分離「已收線 K」與「進行中 K」。

============================================================
3) K 棒運算（OHLC）核心規則
============================================================
即時報價輸入：
- symbol
- regularMarketPrice（最新成交價）
- regularMarketTime（epoch 秒）

對每個 (symbol, interval)：
1. 由 regularMarketTime 計算 bucketStart。
2. 若此 bucket 既有資料列：
   - open  = 不變
   - high  = max(目前 high, 最新價)
   - low   = min(目前 low, 最新價)
   - close = 最新價
3. 若此 bucket 尚無資料列：
   - open = high = low = close = 最新價

以上為標準即時 OHLC 行為。

============================================================
4) Bucket 對齊規則
============================================================
所有 bucket 計算一律在 UTC（儲存層）完成，前端再轉使用者本地時區顯示。

範例：
- 1m   => 向下取整到分鐘
- 5m   => 分鐘向下取整到 5 的倍數
- 15m  => 分鐘向下取整到 15 的倍數
- 30m  => 分鐘向下取整到 30 的倍數
- 1h   => 分/秒/奈秒歸零
- 4h   => 小時向下取整到 4 的倍數，分/秒/奈秒歸零
- 1d   => UTC 00:00
- 1wk  => 週一 UTC 00:00（或依你的商業規則）
- 1mo  => 每月第一天 UTC 00:00
- 3mo  => 每季第一天 UTC 00:00

重點：
- bucket 計算邏輯集中在單一函式。
- 更新路徑與 merge 路徑使用同一套邏輯。

============================================================
5) 資料庫設計
============================================================
資料表：last_candle

建議欄位：
- id（主鍵）
- symbol
- interval
- bucket_start（timestamp）
- open
- high
- low
- close
- updated_at（可選）

建議唯一索引（保留每個 bucket 模式）：
- UNIQUE(symbol, interval, bucket_start)

替代模式（單列覆寫模式）：
- UNIQUE(symbol, interval)
- bucket 輪替時覆寫同一列。
- 注意：若沒有額外機制，頁面刷新後會失去每個 bucket 的即時連續性。

為了圖表品質，建議：
- 使用 bucket 粒度唯一鍵（symbol, interval, bucket_start）。

============================================================
6) 後端 API 規劃
============================================================
保留既有：
- GET /stockdata?symbol=...&interval=...

新增：
- GET /lastcandle?symbol=...&interval=...
  回傳該 symbol/interval 最新的「進行中」K 棒。

可選增強：
- 同時回傳「進行中 K」與「前一根已收線 K」，讓切換更平滑。

============================================================
7) 排程（Scheduler）規劃
============================================================
沿用現有排程頻率（例如：市場開盤時每 5 秒）。

流程：
1. 批次抓取即時報價。
2. 對每個 symbol 與每個支援 interval：
   - 計算 bucketStart
   - upsert 到 last_candle
3. 使用 lock/atomic flag 避免排程重疊。

交易時段：
- 可沿用目前的美股交易時段判斷。

============================================================
8) 前端規劃（CandlestickView）
============================================================
要解決的問題：
- 輪詢時避免整包歷史資料重載。

規劃：
1. 圖表初始化（symbol+interval）：
   - 呼叫 /stockdata 一次
   - 呼叫 /lastcandle 一次
   - merge 成顯示序列
2. 啟動 /lastcandle 輪詢（每 2-5 秒）
3. 每次輪詢：
   - 若 bucketStart 與當前最後一根相同：替換最後一筆
   - 若 bucketStart 為新值：append 新最後一根（或按 merge 規則輪替）
4. 切換 symbol/interval：
   - 停掉舊 timer
   - 清空暫存狀態
   - 重新初始化
5. 元件 unmount：
   - 清除 timer
   - 清除暫存 cache

============================================================
9) Merge 規則（歷史 + 即時）
============================================================
給定 historySeries 與 liveCandle：
- 若 liveCandle.dateTime == history 最後一筆 dateTime：
  用 live 覆蓋最後一筆
- 否則：
  將 live 追加在尾端
- 渲染前一律依 dateTime 升序排序。

避免產生重複 timestamp。

============================================================
10) 時區策略
============================================================
儲存/計算：
- bucket 與 DB timestamp 一律使用 UTC

顯示：
- 使用瀏覽器本地時區
- 可在圖表標題/十字線標籤顯示時區縮寫

後端 bucket 計算不可混入本地時區。

============================================================
11) 效能與穩定性
============================================================
- 前端輪詢時不要每次都呼叫 updateStock/歷史 API。
- 只輪詢輕量的 /lastcandle。
- 視需要加入短效後端快取。
- 圖表採增量更新（替換最後一點）。

============================================================
12) 風險與邊界情況
============================================================
- 即時報價暫時為 null/缺值：略過該 symbol 當次更新。
- 收盤或暫停交易：close 可能停止變動，維持最後可得值。
- DST 變化：只要後端 bucket 用 UTC，風險可控。
- 上游 interval 粒度不一致：應保存「請求的 interval key」，不要依賴上游粒度。

============================================================
13) 驗證清單（未來實作時）
============================================================
Backend：
- 同 bucket：row 數不變，high/low/close 更新正確。
- 新 bucket：新增一筆 bucket row（或單列模式下覆寫）。
- /lastcandle 可回傳正確 symbol+interval 最新值。

Frontend：
- 初次載入：歷史只出現一次。
- 輪詢中：最後一根可更新且不整包重刷。
- 切換 symbol/interval：無舊 timer 或殘留狀態。
- 刷新頁面：可正常重新初始化。
